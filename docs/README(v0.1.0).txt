Состояние проекта (v0.1.0)
Общее описание
Проект — Android-приложение для отслеживания темпа движения (pace) при беге и ходьбе.
Приложение ориентировано на корректную работу с GPS-данными, стабильный расчёт темпа и минималистичный интерфейс.

Реализовано и работает
1. Геолокация
	Используется FusedLocationProviderClient.
	Обновление координат происходит раз в секунду.
	Геолокация работает через foreground service, что предотвращает остановку системой.

2. Foreground-сервис
	Реализован отдельный LocationService.
	Сервис:
		запускается после получения разрешений;
		работает в фоне как foreground service;
		корректно останавливается.
		При остановке освобождаются все ресурсы (GPS, MediaSession).

3. Уведомление
	Постоянное уведомление, которое нельзя смахнуть.
	Используется MediaStyle (как у музыкальных плееров).
	Уведомление:
		не издаёт звуков при обновлении;
		отображается всё время активного трекинга.
		Кнопки управления (старт/стоп) заложены архитектурно, пока реализованы как заглушки.

4. Расчёт темпа (pace)
	Темп отображается в формате минуты на километр (мм:сс).
	Расчёт основан на скорости из GPS.
	Реализована фильтрация данных:
	игнорируются точки с плохой точностью GPS;
	отбрасываются нефизичные скачки скорости;
	учитывается режим активности (бег / ходьба).
	Используется EMA-фильтр (экспоненциальное сглаживание):
	параметры фильтра зависят от точности GPS;
	для бега и ходьбы используются разные коэффициенты;
	фильтр сбрасывается при старте трекинга и смене режима.

5. Режимы активности
	Поддерживаются режимы:
	RUNNING (бег)
	WALKING (ходьба)
	Режим влияет на:
		допустимый диапазон скорости;
		агрессивность сглаживания темпа.
		Смена режима возможна только при выключенном трекинге.

6. Хранилище состояния
	Вся логика сосредоточена в LocationRepository.
	Состояние доступно через StateFlow:
	текущий темп;
	точность GPS;
	режим активности;
	флаг активного трекинга.
	UI подписывается на состояние и не содержит бизнес-логики.

7. Пользовательский интерфейс
	UI реализован на Jetpack Compose + Material 3.
	Основные экраны:
		экран тренировки (рабочий);
		экран истории (заглушка);
		экран настроек (в разработке).
	Используется нижняя навигация.
	Экран тренировки отображает:
		текущий темп;
		статус GPS-сигнала;
		кнопки управления;
		переключатель режима активности.

8. Разрешения
	Корректная работа с разрешениями:
	геолокация;
	уведомления (Android 13+).
	Повторный запрос не выполняется, если разрешения уже выданы.

9. Отладка
	Реализован TelemetryLogger:
	выводит отладочную информацию в Logcat;
	пишет простой текстовый лог в файл;
	фиксирует состояние батареи.
	Используется только для диагностики и анализа поведения приложения.

10. Wear OS
	Приложение для часов уже существует.
	На часы передаётся текущий темп.
	На данный момент:
	отображается одно числовое значение;
	обновление происходит с задержками;
	интерфейс часов минимальный и не оформлен.
	Реализация используется для проверки передачи данных и архитектуры.
11. Текущий статус и ограничения версии

	Данная версия приложения является первой рабочей версией (v0.1.0) и имеет ряд известных ограничений и особенностей.
	Рабочее состояние
	Основной сценарий работы приложения функционирует:
		запуск трекинга;
		получение GPS-данных;
		расчёт и отображение темпа;
		работа foreground-сервиса и уведомления(пока кривых).
	Темп считается корректно и стабильно в целом, с применением сглаживания.
	Архитектура приложения заложена с расчётом на дальнейшее расширение.
	Известные ограничения и недостатки
	Темп даже с текущим EMA-сглаживанием может незначительно колебаться:
	требуется более продвинутый фильтр или комбинированный алгоритм сглаживания;
	текущий фильтр является компромиссом между отзывчивостью и стабильностью.
	При остановке движения требуется примерно 2–3 секунды, чтобы фильтр:
		распознал отсутствие движения;
		корректно показал остановку темпа.
	Интерфейс Wear OS:
		Почти нет
	Экран истории и часть настроек находятся в стадии заготовок и не завершены.
	Локализация приложения выполнена не полностью  
		часть строк интерфейса пока не вынесена в ресурсы  
		присутствуют жёстко заданные строки в коде  
		Поддерживаются только два языка  
			русский  
			английский  

	Планы развития
		Улучшение алгоритма фильтрации темпа:
		более стабильное поведение на малых скоростях;
		более быстрое определение остановки.
		Расширение режимов активности:
			спринт;
			велосипед;
			возможные дополнительные режимы в будущем.

		Доработка интерфейса часов (Wear OS).
		Расширение функциональности экранов истории и настроек.
		Улутшения функционала уведомлений
