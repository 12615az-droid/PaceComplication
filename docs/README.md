# myPace / Мой темп

**myPace** — нативное Android-приложение для бега и ходьбы, созданное для записи тренировок, контроля темпа и анализа личного прогресса.
Проект разрабатывается как полностью локальное персональное приложение. Для основного сценария работы не требуется регистрация, аккаунт, серверная часть, синхронизация с внешним сервисом или обязательная передача данных куда-либо. Данные тренировок сохраняются на устройстве пользователя.
Основная идея myPace — дать простой и гибкий инструмент для личных тренировок: запуск пробежки, контроль темпа, запись дистанции, сохранение истории и последующий анализ результата.
Текущая версия проекта: **0.2**.  
Целевая релизная версия: **1.0**.
---

## Основная идея

myPace создаётся в первую очередь как личное приложение для бега.

Идея проекта появилась из практического опыта использования готовых беговых приложений. В них есть много полезного, но не всё подходит под личный сценарий тренировок.

Часть функций может быть избыточной: регистрация, внешние сервисы, синхронизация, перегруженный интерфейс и дополнительные разделы, которые не используются во время обычной тренировки.

При этом в готовых аналогах не всегда хватает более гибких персональных настроек. Например, хочется точнее управлять поведением приложения, целями тренировки, фильтрацией данных, голосовыми подсказками и отображением нужной информации во время бега.

Поэтому myPace создаётся как локальное персональное приложение, в котором основной акцент сделан не на универсальности для всех, а на возможности настроить беговой трекер под свой собственные требования.

myPace делает акцент не на базовых функциях бегового трекера, а на тех возможностях, которых часто не хватает в готовых приложениях:

- полностью локальная работа без регистрации, аккаунта, сервера и обязательной синхронизации;
- хранение данных на устройстве пользователя без передачи тренировок во внешние сервисы;
- возможность работы как через Google Location Services, так и через нативный Android-провайдер геоданных;
- более гибкая настройка поведения приложения под личный сценарий тренировки;
- возможность тонкой настройки целей, голосовых подсказок и отображаемых параметров;
- подробное секундное логирование тренировки для анализа и резервного восстановления;
- интеграция с часами wear os и другими блютуз устройсвами.

## Основные экраны

### Экран подготовки тренировки

На экране подготовки пользователь выбирает тип активности: бег или ходьба.

Также на этом экране предусмотрены элементы для будущего выбора целей тренировки и быстрых настроек. Сейчас часть этих возможностей находится в состоянии заготовки, но структура экрана уже подготовлена для дальнейшего развития.

### Экран активной тренировки

Экран активной тренировки отображает основные данные во время занятия:

- время;
- дистанцию;
- текущий темп;
- точность GPS.

На этом же экране находятся кнопки управления тренировкой: пауза, продолжение, остановка и сохранение.

### Экран истории

Экран истории показывает сохранённые тренировки из локальной базы данных.

Сейчас отображаются основные данные:

- дата и время тренировки;
- дистанция;
- продолжительность;
- средний темп;
- тип активности.

Также реализована фильтрация истории по типу активности: все тренировки, бег и ходьба.

### Экран параметров

Экран параметров пока находится на раннем этапе.

В дальнейшем он будет использоваться для настройки поведения приложения, фильтров, уведомлений, голосовых подсказок и других функций.

### Диалог целей тренировки

В приложении уже предусмотрен диалог выбора целей тренировки.

Пока это заготовка под будущую функциональность. В дальнейшем планируется добавить цели по:

- темпу;
- времени;
- дистанции;
- пульсу.

## Технологии

Проект разрабатывается как нативное Android-приложение.

Используемые технологии:

- **Kotlin**;

## Архитектура

Приложение развивается в сторону MVVM-архитектуры.

В проекте уже есть разделение между:

- пользовательским интерфейсом;
- состоянием экранов;
- логикой тренировки;
- работой с геолокацией;
- расчётом темпа;
- локальным хранением данных;
- интеграцией с часами.

Архитектура пока не является полностью завершённой. Часть решений ещё требует доработки, упрощения и более строгого разделения ответственности.

Одна из ближайших целей проекта — привести архитектуру к более аккуратному, понятному и поддерживаемому виду.

## Хранение данных

Для хранения тренировок используется комбинированный подход.

Локальная база данных **Room** хранит краткую информацию о тренировке:

- уникальный ID тренировки;
- дату и время;
- тип активности;
- общую дистанцию;
- продолжительность;
- средний темп.

Подробные значения тренировки не записываются напрямую в базу данных. Это сделано осознанно: запись данных каждую секунду в базу была бы избыточной.

Для подробной информации используется отдельный лог-файл. Он связан с записью в базе данных через общий сгенерированный ID тренировки.

В лог записываются данные примерно каждую секунду. Такой подход позволяет:

- хранить подробную информацию о тренировке;
- использовать данные для будущего экрана подробностей;
- анализировать темп и геоданные;
- строить маршрут на карте;
- реализовать резервное восстановление тренировки после аварийного завершения приложения.

Сейчас лог хранится в формате **JSON Lines**. В будущем возможен переход на бинарный формат для экономии места.

## Геолокация и расчёт темпа

В проекте реализованы две версии провайдера геоданных:

- нативный Android-провайдер;
- провайдер на базе Google-сервисов.

Благодаря этому приложение может работать как на устройствах с Google Play Services, так и на более чистых Android-системах без обязательной зависимости от Google.

Дистанция сейчас рассчитывается по GPS-точкам. На данном этапе используется базовый механизм без сложной фильтрации маршрута.

Темп рассчитывается на основе скорости, которую предоставляет геолокационный провайдер. Перед отображением значение проходит через **EMA-фильтр** с настраиваемым коэффициентом `alpha`, а также через простую фильтрацию грубых ошибочных значений.

На текущем этапе фильтрация базовая. В дальнейшем планируется улучшить математику расчёта темпа и сделать фильтры более персонализированными.

При тестировании на открытой местности и хорошем телефоне отличие по дистанции от Nike Run Club составило около одного процента.

## Уведомление активной тренировки

Для активной тренировки используется системное уведомление с кнопками управления.

Текущая реализация построена через подход, похожий на медиаплеер. Это даёт удобные системные кнопки управления тренировкой и может быть полезно для будущей реализации голосовых подсказок.

Решение пока не считается окончательным. В дальнейшем возможно появление второй реализации уведомления и выбор варианта через настройки приложения.

## Интеграция с часами

В проекте начата интеграция с часами.

Сейчас приложение на часах работает как дополнительный экран к тренировке на телефоне. Оно запускается вместе с тренировкой на телефоне и дублирует основные данные, в первую очередь текущий темп.

Часы не управляют тренировкой на телефоне и не завершают её самостоятельно. При завершении тренировки на телефоне приложение на часах также завершает отображение тренировки.

Интеграция пока находится на раннем этапе и требует дальнейшей доработки.

## Статус проекта

myPace находится в активной разработке.

Проект уже имеет рабочую основу: тренировки можно запускать, записывать и сохранять. При этом приложение ещё требует доработки архитектуры, интерфейса, фильтрации данных и подробного отображения сохранённых тренировок.

Главная цель проекта — создать удобное, лёгкое и гибкое приложение для бега, которое работает локально, не требует регистрации и решает конкретные личные задачи без лишней перегруженности.
